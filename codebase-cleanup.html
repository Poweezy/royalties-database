Mining Royalties Database - Codebase Cleanup Guide

This document explains the codebase cleanup process for the Mining Royalties Manager application.

Quick Cleanup Instructions

To clean up the codebase, you can use one of these methods:

  1. Run the PowerShell script: .\cleanup-redundant.ps1
  2. Run the batch file: cleanup-redundant.bat

These scripts will move redundant files to a backup directory called redundant-files-backup.

Core Required Files

These files are essential for the application to function properly:

File Purpose
royalties.html Main entry point and HTML container
app.js Core application logic
js/module-loader.js Dynamic component loading system
js/chart-manager.js Chart creation and management
js/component-initializer.js Ensures components are properly initialized
js/sidebar-manager.js Validates and manages sidebar sections
js/diagnostics.js Helps diagnose and troubleshoot issues
fix.js Consolidated fixes for navigation issues
components/* HTML templates for various sections
css/* Stylesheets for the application

After Cleanup

After running the cleanup script:

  1. Test the application to make sure everything still works correctly
  2. If all works well, you can delete the redundant-files-backup directory
  3. If you encounter issues, you can restore any needed file from the backup directory

Benefits of Cleanup

  • Smaller codebase size
  • Better maintainability with fewer files to track
  • Clearer organization with less duplication
  • Improved loading performance
  • Fewer potential points of failure
