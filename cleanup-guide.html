Mining Royalties Codebase Cleanup Guide

This guide helps identify and remove redundant files to keep the codebase clean and maintainable.

Note: The cleanup scripts will move files to a backup directory, not delete them. This allows you to restore files if needed.

Backup Files (Safe to Remove)

These are backup files of app.js that are no longer needed:

  • app.js.backup
  • app.js.fixed
  • app.js.before-fix
  • app.js.bak
  • app.js (keep this)

Chart Manager Duplicates (Safe to Remove)

These files delegate to js/chart-manager.js, which is the main implementation:

  • chart-manager-v2.js
  • chart-manager.js (in root directory)
  • js/chart-manager.js (keep this)

Redundant Fix Scripts (Safe to Remove)

These scripts have been consolidated into fix.js:

  • update-scripts.js
  • update-scripts-improved.js
  • js/app-fixer.js
  • js/module-loader-fix.js
  • js/section-loader-fix.js
  • js/section-navigation-fix.js
  • js/resource-tracker.js
  • fix.js (keep this)

Empty/Unused Files (Safe to Remove)

These files are empty or unused:

  • js/message-handler.js
  • js/utils/stateManager.js
  • js/utils/validation.js

Required Files (Do Not Remove)

These are the core files needed for the application to function properly:

  • app.js
  • royalties.html
  • royalties.css
  • js/chart-manager.js
  • js/module-loader.js
  • js/component-initializer.js
  • js/diagnostics.js
  • js/sidebar-manager.js
  • fix.js
  • favicon.svg
  • manifest.json

How to Clean Up

You can run either of the provided cleanup scripts to move redundant files to a backup directory:

PowerShell: ./cleanup-redundant.ps1

Batch file: cleanup-redundant.bat

Manual Cleanup

If you prefer to clean up manually, delete or move the files marked as "redundant" above.

After cleanup, make sure to test the application to verify that everything still works correctly.
